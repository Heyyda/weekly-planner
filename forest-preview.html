Forest · Dark · inline edit — тот же UX в тёмной теме

UX-решения:
• Одна карточка за раз. Клик на ✎ у другой задачи автоматически сохраняет текущую и разворачивает новую — не будет двух открытых редакторов.
• День: пилюли, не dropdown. «Сегодня» / «Завтра» + 7 дней недели рядом → один клик вместо двух. Активная — filled forest, остальные — outline. Помещается в ширину 460px.
• Время компактное. Две mono-ячейки HH : MM + ✕. Клик на ячейку → маленький popover с числами (не dropdown).
• Удалить — clay-кнопка слева. Визуально «опаснее» других, но не кричит. При клике — сразу отменяемое удаление через undo-toast (уже есть в проекте).
• Анимация раскрытия: 200ms ease-out, лёгкий fade + expand. Другие задачи сдвигаются плавно (FLIP-подобно, но простая CSS-анимация tkinter-friendly: просто pack_propagate срабатывает естественно).
• Сохранение: Ctrl+Enter, клик вне карточки, или кнопка. Esc — отмена без сохранения. Потеря изменений при переключении недели — no-op (автосохранение при уходе).
• Fallback для узких окон: если ширина окна < 380px (редко, у тебя min 320) — inline не влезет, откатываемся на прежний модальный EditDialog (оставлен в секции 5 ниже).

Что в коде: TaskWidget получает новое состояние _edit_mode: bool. При True вместо row с text/actions рендерится inline _build_edit_view(), при False — привычный row. Переключение не пересоздаёт TaskWidget (чтобы drag-n-drop state и hover-bindings не ломались). В DaySection трекаем _editing_task_id: Optional[str] — чтобы «одна карточка за раз». Старый EditDialog оставляем как функцию, вызов только в fallback-случае.

8. Что конкретно меняется в коде проекта

Карта: какие файлы трогаем, что происходит.

themes.py — добавляем forest_light и forest_dark палитры. Ключи сохраняем те же (bg_primary, accent_brand, accent_done, accent_overdue) — тогда ни один виджет не нужно править, только карта цветов меняется.
Маппинг семантики Forest → существующие ключи:
— bg_primary ← #EEE9DC / #161E1A
— bg_secondary ← bg-elevated (#F5F0E3 / #202A24)
— bg_tertiary ← accent-forest-tint (#E2E0D2 / #1B2620) — на нём стоит today и archive banner
— accent_brand ← accent-forest — заменит синий и в today-strip, и в QuickCapture, и в undo-link, и в arrow-hover
— accent_done ← accent-forest (тот же) — done-галочка и accent сольются в один цвет, это фича Forest
— accent_overdue ← destructive-clay
— text_primary/secondary/tertiary — по карте выше

icon_compose.py (overlay) — заменяем захардкоженные OVERLAY_BLUE_* и OVERLAY_RED_* на forest-токены. Убираем градиент (или оставляем очень слабый forest → forest-dim). Pulse для overdue — между cream-квадратом и clay-бейджем.

task_widget.py — ничего не трогаем если маппинг ключей сохранён. Проверим что done-чекбокс (который сейчас accent_done) ← зелёный forest, overdue (accent_overdue) ← clay.

day_section.py, week_navigation.py, edit_dialog.py, quick_capture.py, undo_toast.py, update_banner.py, login_dialog.py — если всё читается через ThemeManager.get(), поменяются автоматически. Если где-то есть хардкод цвета — ловим и правим.

tray.py (меню-бар) — добавляем пункт «Forest» в переключатель тем, делаем дефолтным.

Порядок действий:
1. Добавить палитры forest_light / forest_dark в themes.py, включить как тему
2. Перекрасить overlay icon_compose.py (flat + forest вместо синего градиента)
3. Запустить, пройти все экраны — ловить захардкоженные цвета если остались
4. Если на real-display плашка-cream теряется на светлом столе — сменить на solid-forest вариант (один ключ поменять)
