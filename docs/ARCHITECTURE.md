## Sidebar: как работает боковая панель

```
        Экран пользователя
┌──────────────────────────┐
│                          │║  ← полоска 6px (trigger)
│   Рабочий стол           │║
│   Другие окна            │║
│                          │║
└──────────────────────────┘

При наведении на полоску:
┌──────────────────────────┐
│                    ┌─────┤
│                    │ Неде│  ← панель выезжает (360px)
│                    │ Пн  │
│                    │ Вт  │
│                    │ ... │
│                    └─────┤
└──────────────────────────┘
```

**Реализация:**
- `overrideredirect=True` — окно без рамки
- `topmost=True` — поверх всех окон
- Полная высота экрана, ширина 360px
- В скрытом состоянии: x = screen_width - 6
- В развёрнутом: x = screen_width - 360
- Анимация: `after(12ms)` с шагом 20px
- Win32 `SetWindowPos` для smooth движения

## Синхронизация: optimistic UI

```
Действие пользователя
        │
        ▼
┌─ Применить к LocalStorage (мгновенно) ──────┐
│  cache.json обновлён                         │
│  UI перерисован                              │
└─────────────────────────────────────────────┘
        │
        ▼ (фоновый поток)
┌─ Добавить в pending_changes ────────────────┐
│  {"op": "create_task", "data": {...}}        │
└─────────────────────────────────────────────┘
        │
        ▼ (каждые 30 сек)
┌─ POST /api/sync ─────────────────────────────┐
│  Отправить pending_changes                    │
│  Получить серверное состояние                 │
│  Обновить LocalStorage                        │
│  Очистить pending_changes                     │
└──────────────────────────────────────────────┘
```

**Конфликты**: сервер побеждает (server wins). При оффлайн — задачи работают локально,
при восстановлении сети — pending_changes отправляются, сервер мерджит.

## Зависимости

### Клиент
| Пакет | Назначение |
|-------|-----------|
| customtkinter | UI фреймворк |
| Pillow | иконки, изображения |
| pystray | system tray |
| keyboard | глобальные хоткеи |
| keyring | безопасное хранение JWT |
| requests | HTTP клиент |

### Сервер
| Пакет | Назначение |
|-------|-----------|
| fastapi | HTTP фреймворк |
| uvicorn | ASGI сервер |
| sqlalchemy | ORM |
| python-jose | JWT |
| passlib | хеширование |
