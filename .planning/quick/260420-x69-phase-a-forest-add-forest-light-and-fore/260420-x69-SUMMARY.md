# Phase A Forest: Add forest_light и forest_dark Summary

**One-liner:** Добавлены две новые палитры Forest (cream+бутылочный-зелёный / тёмный-лес) в `PALETTES`, `forest_light` стала дефолтом для свежих установок и fallback'ом `validate()`, tray-меню получило два новых radio-пункта — всё без правок widget-кода и с сохранением backward-compat для старых settings.json.

## What Changed

### `client/ui/themes.py`
- **PALETTES**: добавлены две записи (`forest_light`, `forest_dark`), каждая с 11 ключами verbatim из спецификации. Старые записи `light` / `dark` / `beige` — без изменений.
- **`ThemeManager.__init__`**: default `initial` параметр изменён `"light"` → `"forest_light"`, fallback при неизвестном initial тоже `"forest_light"`.
- **`ThemeManager.set_theme`**: fallback при unknown тематике изменён `"light"` → `"forest_light"` (и warning-сообщение обновлено).
- **Module docstring**: обновлён с "3 темы" → "5 тем" + добавлена ссылка на Forest refactor spec.
- **НЕ тронуто**: `detect_system_theme()`, `ctk.set_appearance_mode()` (line 114), `get()`, `subscribe()`, FONTS.

### `client/ui/settings.py`
- **`VALID_THEMES`**: добавлены `"forest_light"`, `"forest_dark"`.
- **`UISettings.theme` default**: `"light"` → `"forest_light"`.
- **`validate()`**: fallback при invalid theme изменён `"light"` → `"forest_light"` (и warning-сообщение обновлено).
- **Module docstring**: добавлены новые темы в список допустимых значений `theme:`.
- Остальные поля dataclass + validate() логика для `task_style`/`notifications_mode` — без изменений.

### `client/utils/tray.py`
- **`_build_menu`**: в подменю "Тема" после "Бежевая" добавлены 2 новых radio-пункта — "Forest (светлая)" → `forest_light` и "Forest (тёмная)" → `forest_dark`. "Системная" остаётся последним пунктом.
- **`_cb_theme`**: без изменений — принимает любую валидную строку темы.

### `client/tests/ui/test_themes.py`
- **Переименованы**: `test_palettes_has_three_themes` → `test_palettes_has_all_five_themes`; `test_default_theme_is_light` → `test_default_theme_is_forest_light`; `test_unknown_theme_falls_back_to_light` → `test_unknown_theme_falls_back_to_forest_light`. Ассерты обновлены.
- **Добавлены 2 новых теста**: `test_forest_light_palette_exact_hex_per_spec` и `test_forest_dark_palette_exact_hex_per_spec` — проверяют verbatim hex-значения всех ключевых токенов включая intentionally-darker `accent_brand_light` для forest_light.

### `client/tests/ui/test_settings.py`
- **test_defaults**: `s.theme == "light"` → `"forest_light"`.
- **test_validate_resets_invalid_theme**: `s.theme == "light"` → `"forest_light"` (validate() fallback target).
- **test_store_load_with_corrupted_theme_sanitizes**: `loaded.theme == "light"` → `"forest_light"` (migration path тест).
- **НЕ тронуты**: тесты где "dark"/"beige" переданы явно (user-chosen темы).

## Migration Semantics (подтверждено тестами)

| Сценарий | Результат |
|----------|-----------|
| Свежая установка (нет settings.json) | `UISettings().theme == "forest_light"` |
| Legacy save: `theme="light"` | остаётся `"light"` (backward-compat, user choice respected) |
| Legacy save: `theme="dark"` | остаётся `"dark"` |
| Legacy save: `theme="beige"` | остаётся `"beige"` |
| Corrupted save: `theme="garbage"` | `validate()` мигрирует → `"forest_light"` |
| Tray-меню переключение на Forest | работает без изменений логики `_cb_theme` |
