<objective>
Починить DnD: перетаскивание задачи на пустой день (без задач) не срабатывает.

**Корень проблемы:** `DaySection._update_body_visibility` вызывает `pack_forget()` на `_body_frame` при отсутствии задач. DropZone зарегистрирована на этот `_body_frame` — в распакованном состоянии `winfo_width/height = 0`, `DropZone.get_bbox()` возвращает нулевой прямоугольник, `contains()` всегда возвращает False (см. drag_controller.py:43 — `if x2 <= x1 or y2 <= y1: return False`).

**Фикс:** Регистрировать DropZone на `self.frame` (вся карточка дня — включая header), а не на `_body_frame`. Header упакован всегда, bbox ненулевой всегда.

Purpose: Восстановить возможность перемещать задачу на пустой день — ключевая UX-операция недельного планера.
Output: Исправленный DnD, задача двигается на любой день.
</objective>

**Изменение 1 — `client/ui/day_section.py` (рядом с get_body_frame, строка ~110):**

Добавить новый публичный метод СРАЗУ ПОСЛЕ `get_body_frame`:

```python
def get_drop_frame(self) -> ctk.CTkFrame:
    """Фрейм для регистрации DropZone — вся секция дня (header + body).

    Используется вместо get_body_frame(), потому что _body_frame может быть
    скрыт через pack_forget() при пустом дне (см. _update_body_visibility).
    Корневой self.frame упакован в main_window всегда, поэтому имеет
    ненулевой bbox — DnD на пустой день работает корректно.
    """
    return self.frame
```

Не трогать существующий `get_body_frame` — он используется в `_build()` как parent для TaskWidget.
