<objective>
HOTFIX: приложение падает при открытии главного окна с диалогом "параметр задан неправильно" (ERROR_INVALID_PARAMETER, Windows error 87).

Корневая причина: в `client/ui/main_window.py` вызовы `ctypes.windll.user32.GetWindowLongW` / `SetWindowLongW` / `GetParent` / `ReleaseCapture` / `SendMessageW` идут **без явных `argtypes`/`restype`**. На x64 Python ctypes по умолчанию трактует параметры как C `int` (32-bit), а HWND / WPARAM / LPARAM / LONG_PTR на x64 — 64-bit указатели. Результат: Windows получает усечённые/мусорные значения → ERROR_INVALID_PARAMETER.

Purpose: восстановить запуск главного окна, починить WS_EX_TOOLWINDOW (taskbar-hide из quick 260422-sue) и edge-resize (quick 260422-tah), не сломав существующие 28 pytest-тестов.

Output: `client/ui/main_window.py` с типизированными user32 API + заменёнными вызовами в `_apply_borderless` и `_on_edge_press`.
</objective>

  <action>
    Отредактировать `client/ui/main_window.py`:

    **Шаг 1.** Сразу после блока `import` (после строки `logger = logging.getLogger(__name__)` на ~строке 44) добавить module-level блок типизации Win32:

    ```python
    import ctypes.wintypes as wt

    # ---- Win32 user32 signatures (x64 safe) ----
    # Без явных argtypes/restype ctypes на x64 трактует параметры как C int (32-bit),
    # а HWND/WPARAM/LPARAM/LONG_PTR — pointer-sized (64-bit на x64). Результат:
    # Windows получает усечённые значения → ERROR_INVALID_PARAMETER (код 87).
    _user32 = ctypes.windll.user32

    # GetParent(HWND) -> HWND
    _user32.GetParent.argtypes = [wt.HWND]
    _user32.GetParent.restype = wt.HWND

    # GetWindowLongPtrW / SetWindowLongPtrW — x64 ptr-размер (ctypes.c_ssize_t = LONG_PTR).
    # На 32-bit Python эти функции отсутствуют — fallback на LongW (LONG = c_long).
    if hasattr(_user32, "GetWindowLongPtrW"):
        _user32.GetWindowLongPtrW.argtypes = [wt.HWND, ctypes.c_int]
        _user32.GetWindowLongPtrW.restype = ctypes.c_ssize_t
        _user32.SetWindowLongPtrW.argtypes = [wt.HWND, ctypes.c_int, ctypes.c_ssize_t]
        _user32.SetWindowLongPtrW.restype = ctypes.c_ssize_t
        _get_window_long = _user32.GetWindowLongPtrW
        _set_window_long = _user32.SetWindowLongPtrW
    else:
        _user32.GetWindowLongW.argtypes = [wt.HWND, ctypes.c_int]
        _user32.GetWindowLongW.restype = ctypes.c_long
        _user32.SetWindowLongW.argtypes = [wt.HWND, ctypes.c_int, ctypes.c_long]
        _user32.SetWindowLongW.restype = ctypes.c_long
        _get_window_long = _user32.GetWindowLongW
        _set_window_long = _user32.SetWindowLongW

    # ReleaseCapture() -> BOOL
    _user32.ReleaseCapture.argtypes = []
    _user32.ReleaseCapture.restype = wt.BOOL

    # SendMessageW(HWND, UINT, WPARAM, LPARAM) -> LRESULT
    # WPARAM/LPARAM/LRESULT — pointer-sized, ctypes.c_ssize_t покрывает signed LONG_PTR.
    _user32.SendMessageW.argtypes = [wt.HWND, wt.UINT, ctypes.c_ssize_t, ctypes.c_ssize_t]
    _user32.SendMessageW.restype = ctypes.c_ssize_t
    ```

    Если `import ctypes` уже есть (строка 21) — не дублировать, только добавить `import ctypes.wintypes as wt` в секцию стандартных импортов (рядом с `import ctypes`).

    **Шаг 2.** В методе `_apply_borderless` (~строки 340-369) заменить три прямых вызова `ctypes.windll.user32.*` на модульные обёртки:


    ```python
    # было:
    current = ctypes.windll.user32.GetWindowLongW(hwnd, GWL_EXSTYLE)

    ctypes.windll.user32.SetWindowLongW(hwnd, GWL_EXSTYLE, new)
    # стало:

    new = (current & ~WS_EX_APPWINDOW) | WS_EX_TOOLWINDOW
    _set_window_long(hwnd, GWL_EXSTYLE, new)
    ```

    Сохранить остальную логику (`overrideredirect`, withdraw/deiconify цикл, try/except логирование) без изменений.

    **Шаг 3.** В методе `_on_edge_press` (~строки 488-519) убрать локальное `user32 = ctypes.windll.user32` и заменить все `user32.*` вызовы на `_user32.*`:

    ```python
    # было:
    user32 = ctypes.windll.user32
    widget_hwnd = self._window.winfo_id()
    parent_hwnd = user32.GetParent(widget_hwnd)
    # ...
    user32.ReleaseCapture()
    user32.SendMessageW(hwnd, 0x00A1, ht, 0)
    # стало:
    widget_hwnd = self._window.winfo_id()
    parent_hwnd = _user32.GetParent(widget_hwnd)
    # ...
    _user32.ReleaseCapture()
    _user32.SendMessageW(hwnd, 0x00A1, ht, 0)
    ```

    **Почему именно `ctypes.c_ssize_t`:**
    - `c_ssize_t` = signed pointer-sized integer (на x64 = 8 байт, на x86 = 4 байта).
    - Это Python-эквивалент Win32 `LONG_PTR` / `INT_PTR` / `SSIZE_T`, что точно совпадает с типами, которые Windows ожидает для `GWL_EXSTYLE` значений, `WPARAM`, `LPARAM`, `LRESULT` на соответствующем bitness.
    - `HWND` = pointer (`wintypes.HWND` под капотом — `c_void_p`), тоже pointer-sized — `wintypes.HWND` уже корректно определён в ctypes для обоих архитектур.

    **Pitfalls, не задеть:**
    - НЕ трогать логику `overrideredirect(True)` (PITFALL 1 — Win11 DWM, after(100) уже есть в вызывающем коде).
    - НЕ удалить withdraw/deiconify цикл в `_apply_borderless` (нужен для re-apply EX-style когда окно видимо).
    - НЕ сломать fallback `parent_hwnd if parent_hwnd else widget_hwnd` в `_on_edge_press`.
    - НЕ удалить try/except Exception логирование — graceful degradation остаётся.
    - `GetWindowLongPtrW` существует на Windows XP SP1+ → safe на всех поддерживаемых системах, но `hasattr` fallback оставить для 32-bit Python.
